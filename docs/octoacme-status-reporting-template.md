- Infrastructure changes or improvements
- Performance and reliability metrics

### Documentation (Technical Writer)

## Status Report Cadence

### Weekly Reports
- **Audience**: Project team, Product Lead, immediate stakeholders
- **Format**: This template, shared via email or project management tool
- **Owner**: Project Manager (with input from Scrum Master if applicable)

### Monthly Reports
- **Audience**: Executive stakeholders, sponsors, broader organization
- **Format**: Executive summary + key metrics + strategic updates
- **Owner**: Project Manager + Product Manager

### Ad-Hoc Reports
- **Trigger**: Major milestone, critical issue, or stakeholder request
- **Format**: Adapt this template as needed
- **Owner**: Project Manager

---

## Tips for Effective Status Reporting

1. **Be concise**: Stakeholders are busy; prioritize the most important information
2. **Use visuals**: Charts, graphs, and status indicators improve readability
3. **Be honest**: Report issues and risks transparently to enable proactive support
4. **Focus on impact**: Explain why updates matter, not just what happened
5. **Make asks clear**: Explicitly state what you need from stakeholders
6. **Maintain consistency**: Use the same format and schedule to build trust
7. **Collaborate on input**: Gather updates from all roles to ensure comprehensive reporting
8. **Archive reports**: Maintain a history for retrospectives and lessons learned
