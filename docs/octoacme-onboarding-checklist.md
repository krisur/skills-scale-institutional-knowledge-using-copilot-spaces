## Purpose
Provide a standardized checklist to ensure new team members are effectively onboarded to OctoAcme projects, regardless of their role. This addresses the onboarding gap identified in process improvement initiatives.

## Overview
Effective onboarding accelerates time-to-productivity, improves role clarity, and ensures new team members understand project context, tools, and communication norms. This checklist should be completed within the first two weeks of joining a project.

---

## General Onboarding (All Roles)

### Project Context
- [ ] Review Project One-pager (problem statement, goals, success metrics)
- [ ] Understand project timeline, current phase, and upcoming milestones
- [ ] Review project roadmap and release plan
- [ ] Identify key stakeholders and their roles
- [ ] Review Risk Register to understand current risks and mitigations

### Access & Tools
- [ ] Access to project repository and relevant branches
- [ ] Access to project board (e.g., GitHub Projects, Jira)
- [ ] Access to communication channels (Slack, Teams, email lists)
- [ ] Access to documentation repositories and wikis
- [ ] Access to CI/CD systems (read access minimum)
- [ ] Calendar invites for relevant ceremonies and meetings

### Documentation & Processes
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) to understand team structure
- [ ] Review [Execution & Tracking](octoacme-execution-and-tracking.md) for workflows
- [ ] Understand escalation paths in [Risk Management & Communication](octoacme-risks-and-communication.md)
- [ ] Review Definition of Done (DoD) for the project
- [ ] Understand PR conventions and code review process

### Team Introductions
- [ ] Meet with Project Manager for project overview and expectations
- [ ] Meet with Product Manager to understand product vision and priorities
- [ ] Introduction to team in daily standup or team meeting
- [ ] Scheduled 1:1s with key collaborators based on role
- [ ] Buddy/mentor assigned (if applicable)

## Role-Specific Onboarding

### Developer
- [ ] Clone repository and set up local development environment
- [ ] Run build and tests successfully
- [ ] Review coding standards and contribution guidelines
- [ ] Understand branching strategy and deployment process
- [ ] Shadow code review process
- [ ] Pair with senior developer on first small task
- [ ] Meet with DevOps Engineer to understand deployment pipeline
- [ ] Meet with Technical Writer to understand documentation expectations

### Product Manager
- [ ] Review customer feedback and user research findings
- [ ] Understand product analytics and success metrics dashboards
- [ ] Review backlog and current prioritization rationale
- [ ] Meet with Business Analyst to understand requirements process
- [ ] Meet with UX Designer to review design system and user flows
- [ ] Meet with stakeholders to understand business context

### Project Manager
- [ ] Review project charter and governance structure
- [ ] Understand resource allocation and team capacity
- [ ] Review historical velocity and burndown data
- [ ] Meet with Scrum Master (if applicable) to align on facilitation
- [ ] Understand cross-team dependencies and coordination points
- [ ] Review status reporting format and cadence

### QA/Testing
- [ ] Access to test environments and test data
- [ ] Review test plan and test automation framework
- [ ] Understand quality gates and acceptance criteria process
- [ ] Shadow exploratory testing session
- [ ] Meet with Developers to understand testing handoffs
- [ ] Review bug tracking and severity definitions

### Scrum Master
- [ ] Understand team's current Agile maturity and practices
- [ ] Review sprint cadence and ceremony schedule
- [ ] Identify current team impediments and coaching opportunities
- [ ] Meet with Project Manager to align on delivery coordination
- [ ] Review team velocity trends and process metrics
- [ ] Understand team dynamics and collaboration patterns

### UX Designer
- [ ] Access to design tools and design system
- [ ] Review existing user research and personas
- [ ] Understand design review and approval process
- [ ] Review accessibility standards and guidelines
- [ ] Meet with Product Manager to understand user needs
- [ ] Meet with Developers to understand design handoff process
- [ ] Review usability testing process and schedule

### Technical Writer
- [ ] Access to documentation repositories and publishing tools
- [ ] Review documentation style guide and templates
- [ ] Understand documentation review and approval process
- [ ] Meet with Developers to understand feature documentation needs
- [ ] Review release notes process and timeline
- [ ] Identify gaps in current documentation
- [ ] Meet with Product Manager to align on user-facing messaging

### DevOps Engineer
- [ ] Access to infrastructure and deployment systems
- [ ] Review CI/CD pipeline configuration and deployment process
- [ ] Understand monitoring, alerting, and observability setup
- [ ] Review incident response process and on-call rotation
- [ ] Meet with Developers to understand build and deployment needs
- [ ] Review infrastructure-as-code and automation standards
- [ ] Understand capacity planning and scaling strategy

### Business Analyst
- [ ] Access to requirements documentation and process models
- [ ] Review current requirements and acceptance criteria
- [ ] Understand stakeholder landscape and communication preferences
- [ ] Meet with Product Manager to align on requirements process
- [ ] Meet with Project Manager to understand scope and change management
- [ ] Review business process documentation
- [ ] Understand requirements traceability and change control

---

## 30-Day Check-in

### Review with Manager/Project Lead
- [ ] Onboarding checklist completed
- [ ] Clarity on role responsibilities and expectations
- [ ] Productive contributions to project started
- [ ] Relationships established with key collaborators
- [ ] Identified any blockers or support needs
- [ ] Feedback on onboarding process for continuous improvement

---

## Onboarding Owner
- **Project Manager** (or Scrum Master) coordinates overall onboarding
- **Role-specific leads** (e.g., Tech Lead for Developers, Design Lead for UX) own role-specific sections
- **Assigned buddy/mentor** supports day-to-day questions and culture integration
